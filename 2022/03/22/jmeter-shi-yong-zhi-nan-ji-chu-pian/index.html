Jmeter详细的官方文档地址为：https://jmeter.apache.org/usermanual/get-started.html ，有需求的朋友可以在这里进行更深入的研究。

本文将介绍Jmeter的基础使用方法，并以实际的接口为例说明部分常用组件的使用方法。

1. 下载Jmeter

到官网的下载地址 https://jmeter.apache.org/download_jmeter.cgi 下载最新的release版本。我下载时是 apache-jmeter-5.4.1_src.zip。

2. 运行Jmeter

解压下载的压缩包，进入其bin目录，在terminal中输入命令jmeter即可以GUI模式运行Jmeter，如下图所示：

其他的运行方式将会在有需要时再介绍。

3. Jmeter主要组件介绍

要想在Jmeter中运行测试，至少需要包括测试计划（Test Plan）、线程组（Thread Group）和一个或多个采样器（Sampler）。更为详细的介绍如下，或直接查看官方文档。

  1. 测试计划（Test Plan）

    测试计划描述了JMeter运行时将执行的一系列步骤。一个完整的测试计划将由一个或多个线程组、逻辑控制器、样本生成控制器、侦听器、计时器、断言和配置元素组成。

  2. 线程组（Thread Group）

    所有的控制器和采样器必须在线程组下，其他元素，例如监听器，可以直接放在测试计划下，在这种情况下，它们将应用于所有线程组。顾名思义，线程组用于控制JMeter执行测试的线程数，可以在线程组中：设置线程数、设定加速期、设置执行测试的次数。

    每个线程都将完整地执行测试计划，并且完全独立于其他测试线程。多线程用于模拟与服务器应用程序的并发连接。

  3. 控制器（Controllers）

    JMeter有两种类型的控制器：采样器和逻辑控制器。

    采样器用于向服务器发送请求。例如，如果希望JMeter发送HTTP请求，请添加HTTP请求采样器，并在采样器中添加一个或多个配置元素来定制请求。

    逻辑控制器用于控制何时发送请求的逻辑。例如，您可以添加一个IF控制器，满足特定条件时才发送HTTP请求。

  4. 测试片段（Test Fragments）

    测试片段纯粹用于测试计划中的代码重用，它与线程组元素位于测试计划的同一级别。除非有控制器引用它，否则它不会被执行。

  5. 监听器（Listeners）

    监听器将在JMeter运行时收集并展示关于测试用例的运行信息。例如，“图形结果”将响应时间绘制在图表上，“查看结果树”显示采样器请求和响应的详细信息，并可以显示响应的基本HTML和XML表示形式。
    此外，监听器可以将数据定向到一个文件以供以后使用。JMeter中的每个侦听器都提供一个字段来指示要存储数据的文件。还有一个配置按钮，可用于选择要保存的字段，以及是否使用CSV或XML格式。

  6. 定时器（Timers）

    默认情况下，JMeter线程按顺序执行采样器，期间不暂停。这会使Jmeter在短时间内发出大量请求，可能导致服务器崩溃，因此建议向线程组添加一个计时器来增加延迟。

  7. 断言（Assertions）

    通过添加断言可以“测试”应用程序是否返回了预期的结果。可以向任何采样器添加断言。例如，向HTTP请求添加一个断言，以检查文本“”。然后，JMeter将检查HTTP响应中是否存在文本。如果JMeter找不到文本，那么它会将此标记为失败的请求。

  8. 配置元件

    配置元件与采样器密切配合。配置元素只能从放置该元素的树分支内部访问。尽管它不发送请求（HTTP（S）测试脚本记录器除外），但它可以添加或修改请求。

  9. Pre-Processor 和Post-Processor

    Pre-Processor在发出采样器请求之前执行一些操作，post-Processor则在发出采样器请求之后执行一些操作。

  10. 组件执行顺序

    1. Configuration elements
    2. Pre-Processors
    3. Timers
    4. Sampler
    5. Post-Processors (unless SampleResult is null)
    6. Assertions (unless SampleResult is null)
    7. Listeners (unless SampleResult is null)

4. 举个例子

首次打开Jmeter后，我们在右侧看到有一个空白的测试计划，下面我们就从这里开始一步步添加组件，完成Daily Cost的 “登陆 - 新增一笔支出记录” 这一简短的业务流程为例，先在Jmeter中调通接口。

1. 登陆接口

  • 在测试计划下添加线程组，各参数保持默认数值 1 即可

  • 在线程组下添加 取样器 - HTTP请求，根据API文档完成登陆接口

由于接口的Request Body为Json格式，为此我们还需要添加 配置元件 - HTTP信息头管理器，并添加Header Content-Type: application/json:

  • 添加监听器 - 察看结果树 后运行

运行前需要先保存测试计划。点击运行后可在察看结果树页面看到请求是否成功发送及Request和Response的具体数据。

  • 为登陆接口添加断言

根据登陆接口的定义，登陆成功后接口会返回code为100，我们断言code值为100即可。

首先在察看结果树中进行Json语法测试，如下所示，从Text切换到Json Path Tester，然后输入表达式$.code，其中$表示根节点，点击Test，可以看到下方列出了提取结果为Result[0]=100，是我们想要的值。

然后就可以在登陆请求下添加 断言 - JSON断言，并填入上面已经测试通过的表达式，记得要勾选Additionally assert value，在期望值输入框中填入100。

为了更明显得看到断言的效果，我们可以先在HTTP请求中将登陆用户的密码改为错误的再运行，并能在察看结果树中看到断言结果。当测试通过时不展示JSON断言详情。

如果需要同时对多个字段进行断言，只需要添加多个JSON断言组件。

2. 新增一笔支出记录接口

我们要添加第2个接口了，在此之前我们先将配置了登陆接口的HTTP请求的名称修改为“登陆”。然后再在线程组下添加 取样器 - HTTP请求。

    添加 配置元件 - HTTP请求默认值，并在其中填写Web服务器信息，登陆和新增一笔支出记录接口中的Web服务器信息留空即可。这里需要注意下作用范围，如果HTTP请求默认值在测试计划下面，那么该配置会影响测试计划下的所有HTTP请求取样器；如果在线程组中，那么其影响范围是线程组。
